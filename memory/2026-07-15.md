## Сессия 1

Тема: VS Code не подключался к серверу агента (туннель, ошибка WebSocket 1006)

### Сделано
- Диагностика сервера: code-tunnel.service жив и Connected, попытки подключения до сервера не доходили (пустой journalctl)
- Перезапуск туннеля на сервере — не помогло
- Локальные логи VS Code показали: обрыв происходит на локальном мосте 127.0.0.1 (WebSocket 1006 за миллисекунды), сеть/VPN ни при чём
- Найдено зависшее автообновление VS Code 1.128.0 → 1.128.1 (CodeSetup висел в процессах, три попытки установки). Обновление довершилось само, но туннель не ожил
- Проверен VPN v2RayTun (TUN-режим, xraycore на 127.0.0.1:10801) — отключение не помогло
- Проверка через браузер: 127.0.0.1:порт → ERR_CONNECTION_REFUSED. Вывод: установка VS Code повреждена, сломан внутренний механизм локальных соединений
- Обход: настроено подключение по Remote-SSH вместо туннеля (работает)

### Как настроен Remote-SSH (детали)
- На ноутбуке создан ключ ed25519: `C:\Users\dell\.ssh\id_ed25519`, комментарий `denis-laptop`
- Публичный ключ прописан на сервере в `/root/.ssh/authorized_keys`. Нюанс: служебный ключ Beget был без перевода строки в конце — мой ключ приклеился к его строке, сервер отвергал. Исправлено разделением на две строки (sed)
- `C:\Users\dell\.ssh\config`: Host `agent-server` → root@31.129.109.129, ключ, keepalive
- Установлено расширение ms-vscode-remote.remote-ssh
- В settings.json добавлено: `remote.SSH.useExecServer: false` (без этого зависало на «Инициализация VS Code...»), `remote.SSH.showLoginTerminal: true`, `remote.SSH.connectTimeout: 60`
- Установлен PuTTY (plink.exe) — для SSH-команд с паролем из скриптов

### Решения
- Подключение к серверу теперь через Remote-SSH: F1 → «Remote-SSH: Connect to Host» → agent-server → папка /home/agent/workspace
- Туннель (Remote Explorer → Tunnels → agent) на этом ноутбуке остаётся сломан — виновата побитая установка VS Code
