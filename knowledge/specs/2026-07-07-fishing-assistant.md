# Fishing Assistant — Техническое задание

**Дата:** 2026-07-07
**Автор:** Денис
**Статус:** Draft

---

## Executive Summary

Персональный AI-помощник рыболова. Работает через Telegram. Помогает в двух направлениях: анализирует конкретный водоём (какая рыба есть, как ловить) и подбирает водоём под желаемую рыбу. Учитывает сезон, погоду, тип ловли. Архитектура рассчитана на последующую монетизацию.

---

## Проблема

Сейчас рыболов собирает информацию из разных источников: форумы, приложения, советы знакомых. Единого инструмента, который объединяет данные о водоёме, рыбе, снастях, погоде и времени — нет. Принимать решение «куда ехать и с чем» приходится вручную и долго.

---

## Критерии успеха

- За 2-3 сообщения получить конкретный ответ: «На реке Обь сейчас: щука, судак, окунь. Лучшее время — утро. Снасть для щуки: спиннинг, воблер 7-12 см, проводка равномерная»
- Если нужная рыба в водоёме не водится — прямо сказать об этом и предложить альтернативу
- Работает с телефона, пока стоишь у воды

---

## Два основных сценария

### Сценарий А: «Что есть в этом водоёме?»

1. Пользователь называет водоём (название или GPS)
2. Бот возвращает список рыб, которые там водятся
3. Пользователь выбирает интересующую рыбу (или спрашивает по всем)
4. Бот выдаёт: лучшее время, место в водоёме, снасть, приманку с учётом текущего сезона и погоды

### Сценарий Б: «Где поймать конкретную рыбу?»

1. Пользователь называет рыбу + регион (или текущую геопозицию)
2. Бот подбирает подходящие водоёмы в радиусе
3. Для каждого водоёма — краткая сводка: расстояние, условия, лучшее время
4. Пользователь выбирает водоём → переходит к сценарию А

## Функциональные требования

### P0 — обязательно для MVP

- **Поиск по названию водоёма** — распознать «река Обь», «Байкал», «Можайское водохранилище»
- **Поиск по GPS** — определить ближайшие водоёмы по координатам
- **База рыб по водоёмам** — какая рыба водится, в какой сезон активна
- **Рекомендации по снастям** — тип ловли (спиннинг приоритет), удилище, катушка, леска/шнур, приманка
- **Учёт сезона** — весна/лето/осень/зима меняют рекомендации кардинально
- **Учёт текущей погоды** — давление, температура воды/воздуха, ветер влияют на клёв
- **Рекомендация по времени суток** — утро/день/вечер/ночь для каждой рыбы
- **Рекомендация по месту** — где именно в водоёме искать: перекат, омут, прибрежная зона, русло
- **Отказ с альтернативой** — если рыбы нет в водоёме, сказать об этом и предложить где есть

### P1 — важно, но не в MVP

- **История запросов** — бот помнит, куда ездил пользователь
- **Избранные водоёмы** — сохранить «мои места»
- **Уведомления** — «завтра хороший клёв на Волге, Вы там рыбачили»
- **Фото рыбы** — пользователь присылает фото, бот определяет вид
- **Лунный календарь** — учёт лунных фаз

### P2 — для монетизации

- **Мультипользовательский режим** — каждый со своим профилем
- **Конкретные марки снастей** — «Shimano Nexave 2500, шнур PE 0.10»
- **Партнёрские ссылки** — переход в магазин с рекомендованным товаром
- **Платные отчёты** — детальный PDF-план рыбалки
- **Pro-подписка** — расширенная база, без рекламы

## Технический стек

### Бот
- **Grammy.js** (Node.js) — уже использован в проекте jarvis-architect
- **Telegram Bot API** — интерфейс

### AI-ядро
- **Claude API** (claude-sonnet-5 или claude-haiku-4-5 для скорости) — вся логика рекомендаций, обработка запросов на естественном языке

### Данные о водоёмах
- **OpenStreetMap / Overpass API** — поиск водоёмов по названию и координатам (бесплатно, покрывает всю Россию)
- **База знаний по рыбам** — создать собственный JSON/PostgreSQL-справочник: какая рыба в каком типе водоёма, сезонность, снасти

### Погода
- **OpenWeatherMap API** — текущая погода и 5-дневный прогноз (есть бесплатный tier)
- Параметры: температура воздуха/воды, давление, ветер, облачность

### Хранилище
- **PostgreSQL** — пользователи, история, избранные водоёмы (для монетизации)
- **Redis** — кеш погодных запросов и ответов Claude (экономия токенов)

### Деплой
- VPS (можно тот же, где будет Агент)
- Node.js процесс под PM2
- Переменные окружения: BOT_TOKEN, ANTHROPIC_API_KEY, OPENWEATHERMAP_KEY

## Архитектура запроса

```
Пользователь → Telegram
    ↓
Grammy.js (роутинг, парсинг)
    ↓
Модуль геолокации (OSM/Overpass)  →  Данные о водоёме
Модуль погоды (OpenWeatherMap)    →  Текущая погода
База знаний (рыбы + снасти)       →  Контекст
    ↓
Claude API (промпт с контекстом)
    ↓
Форматированный ответ → Telegram
```

## Данные: что нужно собрать

### База рыб (создать вручную или парсить)

Для каждой рыбы:
- Типы водоёмов где водится (река, озеро, водохранилище, пруд)
- Ареал в России (регионы)
- Сезонная активность (месяц → активность: высокая/средняя/нет)
- Предпочтения: глубина, течение, температура воды
- Тип ловли: спиннинг / поплавок / донка / зимняя
- Приманки по сезонам
- Время суток

Приоритет видов для MVP (спиннинг, Россия):
- Щука, судак, окунь, жерех, голавль, язь, форель, хариус

🌤 Сейчас: +24°C, давление 755 мм рт.ст., слабый ветер

• Удилище: 2,1–2,4 м, тест 10–30 г
• Катушка: 2500–3000

• Приманка: воблер 7–12 см, цвет натуральный
• Проводка: равномерная, твитчинг

⚠️ Нерестовый запрет снят. Лицензия не нужна.

## Out of Scope (не делаем сейчас)

- Мобильное приложение (iOS/Android) — только Telegram
- Платёжная система — только на этапе монетизации
- Соцсеть рыболовов / отчёты / фото уловов
- Карта со слоями — только текстовые ответы

## Открытые вопросы для разработки

1. Откуда брать данные о рыбах в конкретных водоёмах России? OpenStreetMap даст геометрию, но не ихтиофауну. Варианты: парсинг рыболовных форумов, ручное создание базы, Claude генерирует из общих знаний.
2. Как идентифицировать «малые» водоёмы — пруды без названия? Принимать GPS и находить ближайший объект OSM типа water.
3. Температура воды — OpenWeatherMap не даёт. Нужен отдельный источник или рассчитывать приблизительно по воздуху и сезону.
4. Лицензии и нормы вылова по регионам — юридически значимая информация, нужна ли и откуда брать.
