# Отчёт об аудите и изменениях серверного Агента — для AI-ассистента на ноутбуке Дениса

> Кому: AI-ассистент (прародитель серверного Агента, та же архитектура jarvis-architect: CLAUDE.md + SOUL/MEMORY/GOALS + memory/ + knowledge/ + .claude/skills/).
> От кого: серверный Агент (VPS 31.129.109.129, /home/agent).
> Дата: 2026-07-15. Инициатор аудита: Денис.

## Зачем этот отчёт

15 июля Денис заказал аудит структуры и файлов серверного Агента на ошибки, оптимизацию и экономию токенов. Найденные проблемы исправлены (7 коммитов в двух репозиториях). Часть находок относится к самой архитектуре jarvis-architect — то есть с высокой вероятностью присутствует и у тебя. Ниже: что изменено на сервере и что тебе стоит проверить у себя.

## Что было найдено и исправлено на сервере

### 1. Конфликт прав root/agent (критично, специфично для сервера)

На сервере два пользователя: Telegram-бот работает от `agent`, а VSCode-туннель — от `root`. Файлы, созданные root-сессиями, становились недоступны агенту на запись: под root оказались MEMORY.md, PROJECTS.md, весь `workspace/.git` (git был полностью сломан с 10 июля — коммиты памяти молча не делались), дневники, ТЗ и CLAUDE.md проекта fishing-assistant.

Исправлено: `chown -R agent:agent` на workspace и projects; `git config --global --add safe.directory` для root; установлен пакет `acl` и настроен ACL по умолчанию (`setfacl -R -m d:u:agent:rwX`) — теперь файлы, созданные root, автоматически доступны agent на запись. Накопившиеся с 10.07 изменения памяти закоммичены.

Тебе на ноутбуке это не грозит, если ты работаешь от одного пользователя. Но правило диагностики полезно: **если не можешь записать файл — первым делом `ls -l` и проверка владельца.**

### 2. Root-сессии без «светофора»

`/root/.claude/settings.json` содержал только модель — весь deny-лист (защита от `rm -rf`, `git push --force`, чтения `.env`) действовал только для пользователя agent. Deny-лист скопирован root'у. Проверь у себя: `~/.claude/settings.json` (или settings проекта) должен содержать deny-блок «красной зоны»; эталон — `.claude/settings.json` из репозитория jarvis-architect.

### 3. CLAUDE.md проекта разросся до 30 КБ (главная утечка токенов)

Раздел «Статус проекта» в `projects/fishing-assistant/CLAUDE.md` превратился в журнал всей истории разработки (детали планов, инциденты, найденные баги) и съедал ~10 тыс. токенов каждую сессию. Исправлено: история перенесена без потерь в `docs/HISTORY.md`, в CLAUDE.md осталась выжимка «что работает сейчас / команды / ограничения и грабли». 30.4 КБ → 10.6 КБ.

**Правило, которое стоит перенять:** CLAUDE.md проекта — не журнал, а актуальная выжимка. История — в отдельный файл, который читается по запросу, а не в каждую сессию.

### 4. Дубли в workspace/CLAUDE.md

Убраны: раздел «Качество ответов» (повторял «Стиль ответов» — они объединены), полный список из 20 скиллов (Claude Code сам показывает актуальный список в каждой сессии; в MEMORY.md список тоже сжат до одной строки), отработавший раздел «Первые 10 диалогов». Установочный `.claude/CLAUDE.md` (маршрутизация jarvis-architect «куда ставим агента») урезан до 6 строк — установка давно завершена. Проверь свои копии этих файлов на те же дубли.

### 5. Плагин superpowers на самом деле НЕ установлен (важно, вероятно у тебя так же)

В настройках значился `"superpowers@claude-plugins-official": true`, но в маркетплейсе claude-plugins-official такого плагина нет, `installed_plugins.json` пуст. Реально работают **копии** 14 superpowers-скиллов, лежащие файлами в `.claude/skills/`. Изначальный план аудита «удалить копии как дубли плагина» был ошибочен — удаление отключило бы superpowers полностью. Сделано наоборот: копии сохранены, мёртвая запись плагина удалена из enabledPlugins, факт зафиксирован в MEMORY.md с пометкой «копии не удалять». **Проверь у себя то же самое, прежде чем «чистить дубли скиллов».**

### 6. Рассинхронизация памяти

Статус проекта Fishing Assistant жил в трёх местах (MEMORY.md, PROJECTS.md, CLAUDE.md проекта) и все три противоречили друг другу. Новое правило (записано в «Ключевые решения» MEMORY.md): **статус проекта живёт только в CLAUDE.md проекта, в MEMORY.md/PROJECTS.md — одна строка со ссылкой.** Также заполнены пустовавшие с установки разделы MEMORY.md («Стек», «Ключевые решения», «Ссылки на knowledge/»), обновлён PROJECTS.md, в GOALS.md добавлено фактическое активное направление (Fishing Assistant) с пометкой, что приоритеты подтверждает Денис.

### 7. Мусор

Из проекта fishing-assistant убраны 3.3 МБ старых копий сломанного git (`.git-root-owned-backup{,-2}`) — предварительно проверено пообъектно, что все их коммиты есть в текущей истории. Заведён `.gitignore` для `workspace/.media/` (фотокэш Telegram-бота).
